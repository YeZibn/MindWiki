# 6.3 正文抽取优先级设计

## 任务目标

定义预处理阶段对不同内容类型的优先级划分，明确哪些内容属于正文主干，哪些属于结构辅助、补充说明、噪音候选或阻断内容，为后续 section 切分、chunk 构建、检索权重和回答引用提供统一依据。

本子任务对应路线图中的 `6.3`：

- 明确正文抽取优先级，区分正文、注释、脚注、附录等内容

## 设计目标

正文抽取优先级不是简单决定“保留还是删除”，而是回答以下问题：

- 哪些内容构成文档核心知识主干
- 哪些内容主要用于结构理解
- 哪些内容可以作为补充证据
- 哪些内容应降低召回和生成优先级
- 哪些内容不应进入后续处理链路

第一阶段重点不是激进裁剪，而是建立稳定的内容角色分层。

## 分层原则

### 1. 正文优先

后续检索、回答和引用应优先依赖正文主干，而不是目录、页眉页脚、模板块等边缘内容。

### 2. 保守保留

不确定内容先保留，再通过内容角色和权重控制其影响，而不是在预处理阶段直接删除。

### 3. 可引用优先于美观

如果一段内容可能成为证据，就不应因为“看起来像附属信息”而被提前丢弃。

### 4. 结构与正文分离

标题、目录、导航块对结构理解很重要，但不等于正文段落本身，应单独标记。

## 内容角色分层

第一阶段将内容分为 5 类：

1. `body`
2. `supporting_structure`
3. `supporting_content`
4. `noise_candidate`
5. `invalid_content`

## 1. `body`：正文主干

### 定义

`body` 表示文档中直接承载知识事实、解释、论证、定义、结论或方法步骤的核心内容。

### 典型内容

- Markdown 标题下的正常正文段落
- PDF 正常章节正文
- 与主题直接相关的定义、说明、分析、论证
- 事实型表格
- 正常列表项
- 正常引用块

### 处理原则

- 后续 section / chunk 构建优先围绕 `body`
- 检索时优先召回
- 回答时优先作为核心证据
- 引用时优先展示

### 补充说明

#### 表格

如果表格承载事实信息，例如：

- 指标数据
- 法律条款对照
- 医学对比信息
- 结构化知识摘要

则表格按 `body` 处理，而不是降级为附件或噪音。

#### 引用块

Markdown `blockquote` 或类似引用块，若属于正文论述的一部分，默认按 `body` 处理。

原因：

- 引用块可能包含关键定义、引文、观点或证据
- 第一阶段不将其默认降级为补充信息

## 2. `supporting_structure`：结构辅助内容

### 定义

`supporting_structure` 表示主要用于描述文档组织结构、导航关系和章节骨架的内容。

### 典型内容

- 标题
- 小节标题
- 目录
- 提纲型列表
- Markdown 导航块

### 处理原则

- 保留
- 用于 section 归属和文档结构理解
- 可参与检索，但优先级低于 `body`
- 默认不作为回答主证据

### 补充说明

#### 标题

标题本身不是正文段落，但具有高结构价值。

它的作用包括：

- 决定正文归属
- 帮助 section 切分
- 帮助检索结果展示
- 提供文档结构上下文

因此标题应保留高权重结构地位，但不等于正文证据段落。

#### 目录

目录不算正文，但保留并可低权参与检索。

原因：

- 目录能帮助用户理解文档范围
- 目录可以帮助匹配章节名称
- 但目录通常不应作为事实回答的首选证据

## 3. `supporting_content`：补充说明内容

### 定义

`supporting_content` 表示对正文有补充价值，但通常不构成文档主干表达的内容。

### 典型内容

- 脚注
- 附录
- 注释说明
- 参考补充
- 表格备注
- 辅助解释性附加段落

### 处理原则

- 保留
- 可参与检索
- 可作为补充证据被引用
- 优先级低于 `body`
- 当正文证据不足时，可以作为辅助支撑

### 补充说明

#### 脚注

脚注不默认视为正文主干，但不能被简单视为噪音。

在很多专业资料中，脚注可能包含：

- 条款说明
- 来源限定
- 范围边界
- 条件补充

因此脚注应保留为 `supporting_content`。

#### 附录

附录通常不作为第一优先正文，但可能包含重要背景、定义补充或案例材料。

因此附录默认归入 `supporting_content`，而不是 `noise_candidate`。

## 4. `noise_candidate`：噪音候选

### 定义

`noise_candidate` 表示疑似噪音、重复信息或低可信片段，但第一阶段不一定彻底删除。

### 典型内容

- 页眉页脚
- 重复版权声明
- 模板导航块
- 抽取顺序异常片段
- 乱码片段
- 低质量重复标题

### 处理原则

- 保留或局部移除
- 默认不作为回答主证据
- 检索时降权
- 若命中，应结合其他高质量上下文判断

### 补充说明

#### 页眉页脚

即使某些页眉页脚被保留，也不应优先参与回答构造。

#### 乱码片段

乱码片段默认不应成为独立核心证据，但如果整篇文档中只有局部异常，不应因此抛弃整篇内容。

## 5. `invalid_content`：阻断内容

### 定义

`invalid_content` 表示不应进入后续 section、chunk、检索和生成链路的内容。

### 典型内容

- 空文档
- 标准化后正文几乎为空
- PDF 抽取失败页
- OCR 必需但当前未支持的扫描 PDF
- 无法正确解码的文本

### 处理原则

- 导入失败或标记为不可用
- 不进入后续正文抽取结果
- 不参与检索和回答

## 常见内容类型归类表

| 内容类型 | 第一阶段角色 | 说明 |
| --- | --- | --- |
| 正常正文段落 | `body` | 核心知识主干 |
| 正常列表项 | `body` | 若表达事实、步骤、论点，则视为正文 |
| 事实型表格 | `body` | 承载结构化事实信息 |
| 正常引用块 | `body` | 默认按正文处理 |
| 标题 | `supporting_structure` | 高结构价值，不是正文段落 |
| 目录 | `supporting_structure` | 保留并低权参与检索 |
| 导航块 | `supporting_structure` 或 `noise_candidate` | 取决于是否稳定表达结构 |
| 脚注 | `supporting_content` | 可作为补充证据 |
| 附录 | `supporting_content` | 可补充正文背景与材料 |
| 表格备注 | `supporting_content` | 辅助理解，不是正文主干 |
| 页眉页脚 | `noise_candidate` | 默认降权 |
| 乱码片段 | `noise_candidate` | 默认不作核心证据 |
| 顺序异常片段 | `noise_candidate` | 默认降权 |
| 空内容 | `invalid_content` | 直接阻断 |
| OCR 必需但未支持 | `invalid_content` | 当前阶段失败 |

## 与 section / chunk 的关系

`6.3` 会直接影响后续切分策略：

- `body` 优先形成 section 和 chunk 主体
- `supporting_structure` 主要用于 section 边界和归属
- `supporting_content` 可独立成块，但默认权重低于正文块
- `noise_candidate` 可选择单独成块并降权，避免污染正文块
- `invalid_content` 不进入切分链路

## 与检索层的关系

后续检索层可基于内容角色设计默认权重：

- `body`：最高优先级
- `supporting_structure`：中等偏低优先级
- `supporting_content`：中等优先级
- `noise_candidate`：低优先级
- `invalid_content`：不参与检索

说明：

- `supporting_content` 的优先级通常低于 `body`
- 但在某些专业资料中，它可能比目录更有证据价值

## 与生成层的关系

后续生成层构造回答时，应优先使用：

1. `body`
2. `supporting_content`
3. `supporting_structure`

对于 `noise_candidate`：

- 不应优先作为主证据
- 如被引用，应尽量结合其他证据共同出现
- 如果上下文主要由噪音候选构成，应拒绝给出确定性回答

## 与引用展示的关系

引用展示时建议同时体现：

- 来源文档标题
- section 标题
- 内容角色
- 页码或行号等定位信息

这样可以帮助用户理解：

- 当前证据来自正文还是补充说明
- 该证据是否属于主要依据

## 最终结论

`6.3` 正式建立第一阶段正文抽取优先级分层：

- `body`：正文主干，优先用于检索、回答、引用
- `supporting_structure`：结构辅助内容，保留但不作为主证据
- `supporting_content`：补充说明内容，可作为辅助证据
- `noise_candidate`：噪音候选，默认降权处理
- `invalid_content`：阻断内容，不进入后续链路

其中本阶段重点确认如下：

- 目录不算正文，但保留并低权参与检索
- 脚注和附录不算正文主干，但可作为补充证据
- 表格若承载事实信息，应按正文处理
- 标题不是正文段落，但必须保留高权结构地位
- 引用块默认按正文体系处理

该设计将为后续 `7.x` 切分策略、`8.x` 检索权重和 `10.x` 回答引用规则提供统一语义基础。
