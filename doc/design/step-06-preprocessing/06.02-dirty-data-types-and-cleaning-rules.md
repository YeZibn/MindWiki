# 6.2 脏数据类型与清洗规则设计

## 任务目标

定义第一阶段常见脏数据类型，并为每类脏数据建立统一处理策略，确保预处理链路既能清除高确定性噪音，又不会因为激进清洗破坏引用准确性。

本子任务对应路线图中的 `6.2`：

- 列出常见脏数据类型，并为每类噪音定义清洗规则

## 设计原则

### 1. 优先保护可引用性

清洗规则必须服从项目核心约束：

- 检索成功必须定位到具体部分
- 回答必须带引用
- 宁可保守，不要幻觉

因此：

- 不确定是否为噪音的内容，优先保留
- 即使清洗，也要尽量保留位置映射
- 清洗后若影响定位能力，应记录 warning

### 2. 按处理方式分级

第一阶段将脏数据分为三类：

1. 可直接清洗
2. 只做标记，不轻易删除
3. 直接失败或阻断

这样做的目的是：

- 让清洗策略可解释
- 降低误删风险
- 为后续检索降权和人工介入保留空间

### 3. 清洗规则必须高确定性

只有在规则足够稳定、误判成本较低时，才允许自动删除或替换内容。

例如：

- BOM 可直接删除
- 换行符可直接统一
- 连续空行可规整
- 疑似目录、疑似附录、疑似页眉页脚不应轻易硬删

## 分类一：可直接清洗

这类脏数据具有较高确定性，允许自动处理。

### 1. BOM 与编码噪音

常见表现：

- UTF-8 BOM
- 少量可确定的编码残留符号

处理规则：

- 自动移除 BOM
- 编码统一到 `UTF-8`
- 若编码无法识别，则转入失败或人工介入，而不是强行猜测

### 2. 换行符混乱

常见表现：

- `\r\n`
- `\r`
- 混合换行

处理规则：

- 统一转换为 `\n`

### 3. 多余空白字符

常见表现：

- 行尾多余空格
- 多个连续空格
- 制表符混入普通段落

处理规则：

- 普通正文中规整为单一可接受格式
- 代码块、表格等结构化区域内保守处理

### 4. 连续空行和空段落

常见表现：

- 多个连续空行
- 重复空段落

处理规则：

- 将连续空行规整到合理上限
- 删除明显无信息量的重复空段落

### 5. PDF 断行拼接

常见表现：

```text
这是一个被 PDF 排版强制切开的句子，
下一行仍然是同一段内容。
```

处理规则：

- 对高置信度连续正文行进行拼接
- 标题、列表、表格、脚注区域不做激进拼接
- 拼接后尽量保留原位置映射

### 6. 英文连字符断词

常见表现：

```text
retriev-
al
```

处理规则：

- 对高置信度英文断词进行合并
- 中文场景下不主动做复杂词法修复

## 分类二：只做标记，不轻易删除

这类内容可能是噪音，也可能是有效内容。第一阶段默认保留，并打上结构标记或 warning。

## 1. PDF 页眉页脚

常见表现：

- 每页重复出现的文档标题
- 每页底部页码
- 固定版权信息
- 公司名或机构名

正式策略：

- 高置信度重复项可自动移除
- 低置信度项默认保留
- 无论是否移除，都应记录 `header_footer_candidate` 或相关 warning

说明：

- 这是第一阶段少数允许“有条件自动移除”的边界噪音
- 判断必须基于跨页重复和位置稳定性
- 无法高置信度判断时，应优先保留

## 2. 目录页或目录段

常见表现：

- “目录”
- 多级标题列表
- 标题后带页码

正式策略：

- 默认保留
- 标记为 `toc_candidate`
- 后续检索层可对其降权
- 标准化阶段不硬删

说明：

- 目录本身有时能帮助用户理解文档结构
- 但其检索价值通常低于正文
- 删除应留给更成熟的后续阶段，而非第一阶段

## 3. 脚注与附录

常见表现：

- 页脚注释
- 参考说明
- 附录章节
- 法律条款注释说明

正式策略：

- 默认保留
- 分别标记为 `footnote_candidate` 和 `appendix_candidate`
- 不从正文体系中静默移除

说明：

- 对法律、医学、学习资料来说，脚注和附录可能是重要依据
- 即使后续检索层对其降权，也不应在预处理阶段提前删除

## 4. 乱码片段与异常字符

常见表现：

- PDF 抽取产生大量异常符号
- 非法字符集中出现
- 段落可读性明显受损

正式策略：

- 默认保留原样
- 记录 `garbled_text_candidate` 或低文本质量 warning
- 不对局部段落做静默删除

说明：

- 单段乱码不等于整份文档不可用
- 但应将质量风险传递给后续检索和生成层

## 5. 提取顺序异常段落

常见表现：

- PDF 抽取时段落顺序错乱
- 表格和正文穿插
- 多栏排版被串行拼接

正式策略：

- 保留原始抽取结果
- 记录 `extraction_order_unstable`
- 不在第一阶段尝试复杂重排

说明：

- 复杂重排属于高风险修复
- 第一阶段先保证可追溯，不追求过度纠正

## 6. Markdown 导航性内容

常见表现：

- 自动生成目录
- 模板注释
- Obsidian 中的导航块
- 特定模板标记

正式策略：

- 默认保留
- 可识别常见模式并标记 `navigation_block_candidate`
- 第一阶段不做激进删除

说明：

- 导航性内容与正文边界不总是稳定
- 误删真实内容的代价高于保留少量噪音

## 分类三：直接失败或阻断

这类情况继续清洗意义不大，或会生成不可检索、不可引用的脏数据，应在导入或标准化阶段阻断。

### 1. 文件无法读取

常见表现：

- 路径失效
- 文件损坏
- 权限不足

处理规则：

- 导入失败
- 记录明确错误原因

### 2. 编码完全无法识别

常见表现：

- 文本解码持续失败
- 内容几乎无法解释为有效文本

处理规则：

- 导入失败
- 不强行猜测编码

### 3. PDF 文本抽取为空

常见表现：

- 文本型 PDF 抽取结果为空
- 图片型 PDF 未启用 OCR

处理规则：

- 导入失败
- 记录 `pdf_text_extraction_empty`

### 4. 疑似扫描件但当前不支持 OCR

常见表现：

- 文本密度极低
- 每页几乎无可抽取文本
- 明显为图片扫描件

处理规则：

- 导入失败
- 记录 `ocr_required_but_not_supported`
- 明确提示“当前阶段不支持 OCR，后续会增加”

### 5. 标准化后正文几乎为空

常见表现：

- 除标题、页码、空白外几乎无正文
- 仅剩模板噪音

处理规则：

- 导入失败或标记为不可用
- 默认不进入后续 section / chunk / 检索链路

## 脏数据类型总表

| 脏数据类型 | 示例 | 第一阶段处理方式 |
| --- | --- | --- |
| BOM | UTF-8 BOM | 直接清洗 |
| 换行混乱 | `\r\n` / `\r` 混用 | 直接清洗 |
| 多余空白 | 行尾空格、重复空段 | 直接清洗 |
| PDF 断行 | 正文被硬换行 | 高置信度拼接 |
| 英文连字符断词 | `retriev-` / `al` | 高置信度拼接 |
| 页眉页脚 | 重复标题、版权、页脚 | 高置信度移除，否则标记 |
| 目录页 | 目录与页码列表 | 保留并标记 |
| 脚注 | 页脚注释 | 保留并标记 |
| 附录 | Appendix / 附录 | 保留并标记 |
| 乱码片段 | 异常字符密集 | 保留并 warning |
| 顺序异常 | 多栏错序 | 保留并 warning |
| Markdown 导航块 | 自动目录、模板块 | 保留并标记 |
| 无法读取文件 | 文件损坏 | 失败 |
| 编码无法识别 | 无法解码 | 失败 |
| PDF 抽取为空 | 无正文文本 | 失败 |
| 需要 OCR 但不支持 | 扫描版 PDF | 失败 |
| 清洗后正文为空 | 内容不可用 | 失败 |

## 标记与 warning 建议

建议在标准化阶段增加以下标记或 warning：

- `header_footer_candidate`
- `toc_candidate`
- `footnote_candidate`
- `appendix_candidate`
- `navigation_block_candidate`
- `garbled_text_candidate`
- `extraction_order_unstable`
- `low_text_quality`

说明：

- 这些标记主要用于后续解析、检索降权、生成保守策略和人工排障
- 它们本身不一定阻断导入
- 但如果聚合后说明整文质量过低，可升级为失败

## 与后续检索层的关系

`6.2` 的清洗规则不只影响文本质量，也会直接影响检索策略。

后续检索层可利用这些标记做：

- 目录段降权
- 页眉页脚候选降权
- 脚注和附录单独控制权重
- 低质量文本降低召回优先级

这样可以避免在预处理阶段过度删除内容，同时仍控制检索噪音。

## 与后续生成层的关系

如果一个 chunk 来自以下区域：

- `toc_candidate`
- `header_footer_candidate`
- `garbled_text_candidate`
- `footnote_candidate`

则后续生成层应更保守：

- 降低其作为核心证据的优先级
- 若被引用，应明确显示来源位置
- 若上下文质量不足，应拒绝给出确定性结论

## 最终结论

`6.2` 建立了第一阶段预处理清洗的基本原则：

- 高确定性噪音直接清洗
- 模糊边界内容默认保留并标记
- 会污染知识库或导致不可引用结果的情况直接失败

其中针对本阶段重点讨论结果，正式采用以下策略：

- PDF 页眉页脚：高置信度时可自动移除，否则保留并标记
- 目录页：保留并标记为 `toc_candidate`
- 脚注与附录：保留并标记
- 乱码片段：保留原样并 warning，不静默删段
- Markdown 导航性内容：保留并标记，不激进删除

该设计将为后续 `6.3` 正文抽取优先级、`7.x` 切分规则和 `8.x` 检索降权策略提供依据。
