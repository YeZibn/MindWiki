# 9.6 引用载荷结构设计

## 任务目标

为个人知识库 RAG 项目第一阶段定义统一的引用载荷结构，使进入上下文和答案的证据片段同时满足生成层引用、前端展示和原文跳转的需要。

本子任务对应路线图中的 `9.6`：

- 设计引用载荷结构，确保前端可以直接渲染来源与跳转位置

## 关联文档

- `09.05-context-builder-truncation-and-assembly.md`
- `../step-07-chunking-and-enrichment/07.04-offset-and-position-mapping-format.md`

## 用户确认结论

基于当前讨论，`9.6` 采用以下正式设计：

- 引用载荷分为：
  - 证据标识层
  - 展示层
  - 跳转定位层
- 额外保留：
  - `sub_query_id`
  - `evidence_role`
- `evidence_role` 第一阶段先只分：
  - `primary`
  - `supporting`
- `snippet` 只用于展示，不替代真实证据

## 一、设计边界

### 1. `9.6` 解决的问题

`9.6` 的目标是定义一套统一的 citation payload，使：

- 生成层可以引用
- 前端可以展示
- 用户可以跳转回原文

### 2. `9.6` 不解决的问题

本任务不负责：

- 决定哪些 chunk 进入上下文
- 重新组织上下文内容

这些内容已经由 `9.5` 决定。

## 二、证据标识层

第一阶段建议至少包含：

- `citation_id`
- `chunk_id`
- `document_id`
- `section_id`

### 作用

- 唯一标识这条引用
- 支持回溯到 chunk 和文档

## 三、展示层

第一阶段建议至少包含：

- `document_title`
- `section_title`
- `source_type`
- `snippet`
- `match_sources`

### 1. `document_title`

- 用于前端展示文档来源标题

### 2. `section_title`

- 用于展示该证据来自文档的哪一节

### 3. `source_type`

- 用于展示来源类型，例如：
  - `markdown`
  - `pdf`
  - `web_clip`

### 4. `snippet`

- 仅用于前端展示
- 不替代真实证据

正式策略：

- 真实证据仍然是原始 `chunk_text`

### 5. `match_sources`

- 用于表示该证据主要命中自哪些字段

## 四、跳转定位层

第一阶段建议统一采用：

- `location`

其结构对齐 `7.4` 的位置映射设计。

### Markdown

至少支持：

- `line_start`
- `line_end`

### PDF

至少支持：

- `page_start`
- `page_end`
- `page_block_start`
- `page_block_end`

## 五、额外增强字段

## 1. `sub_query_id`

第一阶段正式保留：

- `sub_query_id`

原因：

- 当前上下文按 `sub-query` 分段组织
- 每条证据应知道它服务于哪个子任务

## 2. `evidence_role`

第一阶段正式保留：

- `evidence_role`

第一阶段先只分：

- `primary`
- `supporting`

### 含义

- `primary`：该子任务的主证据
- `supporting`：补充证据

## 六、建议结构示例

```json
{
  "citation_id": "cit_001",
  "sub_query_id": "sq_1",
  "chunk_id": "chunk_101",
  "document_id": "doc_001",
  "section_id": "sec_008",
  "document_title": "个人知识库 RAG 设计",
  "section_title": "Step 8. 构建索引与召回能力",
  "source_type": "markdown",
  "snippet": "Step 8 主要负责基础索引、召回接口、BM25 与向量混合召回等能力。",
  "match_sources": [
    "section_title",
    "chunk_text"
  ],
  "evidence_role": "primary",
  "location": {
    "line_start": 120,
    "line_end": 134
  }
}
```

## 七、最终结论

`9.6` 第一阶段正式采用以下方案：

- 引用载荷分为：
  - 证据标识层
  - 展示层
  - 跳转定位层
- 证据标识层至少包含：
  - `citation_id`
  - `chunk_id`
  - `document_id`
  - `section_id`
- 展示层至少包含：
  - `document_title`
  - `section_title`
  - `source_type`
  - `snippet`
  - `match_sources`
- 跳转定位层通过 `location` 对齐 `7.4` 的位置映射设计
- 额外保留：
  - `sub_query_id`
  - `evidence_role`
- `evidence_role` 第一阶段先只分：
  - `primary`
  - `supporting`
- `snippet` 只用于展示，不替代真实证据
