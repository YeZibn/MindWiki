# 9.4 子任务级 LLM Rerank 方案

## 任务目标

为个人知识库 RAG 项目第一阶段定义对子任务候选集的 LLM rerank 方案，明确 rerank 的输入单元、判断目标、输入字段、输出结构与截断规则，使每个 `sub-query` 都能得到一组高价值证据候选。

本子任务对应路线图中的 `9.4`：

- 选择 rerank 方案并定义 rerank 的输入输出结构，明确其发生在基础召回融合之后

## 关联文档

- `09.03-subquery-local-merge-strategy.md`

## 用户确认结论

基于当前讨论，`9.4` 采用以下正式设计：

- 对每个 `sub-query` 单独执行 rerank
- rerank 输入单元为：
  - `(sub_query, chunk)` 对
- rerank 目标为：
  - 评估该 chunk 对当前 `sub_query` 的证据价值
- 第一阶段采用：
  - LLM 进行子任务级语义 rerank
- rerank 主输入采用：
  - `sub_query_text`
  - `document_title`
  - `section_title`
  - `chunk_text`
- `fused_rrf_score` 仅作为辅助调试信息保留，不作为 rerank 主判断输入
- 每个 `sub-query` 保留：
  - `top 5`

## 一、设计边界

### 1. `9.4` 解决的问题

`9.4` 的目标是：

- 对每个 `sub-query` 的候选集做更高质量的语义证据排序

### 2. `9.4` 不解决的问题

本任务不负责：

- 子任务间统一排序
- 上下文拼接
- 引用载荷定义

这些内容分别在 `9.5`、`9.6` 中展开。

## 二、rerank 粒度

第一阶段正式采用：

- 对每个 `sub-query` 单独 rerank

这意味着：

- `sq1` 的候选只和 `sq1` 的候选比较
- `sq2` 的候选只和 `sq2` 的候选比较

说明：

- 不在 `9.4` 中做跨 `sub-query` 统一排序

## 三、rerank 输入单元

第一阶段 rerank 输入单元正式定义为：

- `(sub_query, chunk)` 对

这意味着：

- 同一个 chunk 可以在不同 `sub-query` 下分别出现
- 它们可拥有不同的 rerank 分数

## 四、rerank 目标

第一阶段 LLM rerank 的目标是判断：

- 该 chunk 对当前 `sub_query` 是否直接相关
- 该 chunk 是否能作为高价值证据
- 该 chunk 是否信息完整、不是噪音

这不是“好不好看”的排序，而是：

- 证据价值判断

## 五、rerank 输入字段

第一阶段建议 LLM rerank 主输入采用：

- `sub_query_text`
- `document_title`
- `section_title`
- `chunk_text`

### 第一阶段暂不作为主判断输入的字段

- `fused_rrf_score`
- `match_sources`
- 其他过多系统元数据

说明：

- 这些字段可保留在结构中用于调试
- 但不作为第一阶段主语义判断输入，避免干扰

## 六、rerank 输出

第一阶段建议每条候选至少输出：

- `rerank_score`
- `rerank_reason`

### `rerank_score`

- 表示 LLM 对该 chunk 证据价值的综合评分

### `rerank_reason`

- 表示该评分的简短解释

## 七、截断规则

第一阶段正式采用：

- 每个 `sub-query` 保留 `top 5`

说明：

- `top 5` 是 rerank 后保留给 `9.5` context builder 的候选规模

## 八、建议输出结构

```json
{
  "sub_query_id": "sq_1",
  "sub_query_text": "Step 8 的职责边界是什么？",
  "reranked_candidates": [
    {
      "chunk_id": "chunk_101",
      "rerank_score": 0.93,
      "rerank_reason": "该片段直接定义了 Step 8 的职责范围，且信息完整。",
      "fused_rrf_score": 0.0213,
      "document_id": "doc_001",
      "section_id": "sec_008",
      "document_title": "个人知识库 RAG 设计",
      "section_title": "Step 8. 构建索引与召回能力",
      "chunk_text": "Step 8 主要负责基础索引、召回接口、BM25 与向量混合召回等能力。"
    }
  ]
}
```

## 九、最终结论

`9.4` 第一阶段正式采用以下方案：

- 对每个 `sub-query` 单独执行 rerank
- rerank 输入单元为：
  - `(sub_query, chunk)` 对
- rerank 目标为：
  - 评估该 chunk 对当前 `sub_query` 的证据价值
- 第一阶段采用：
  - LLM 进行子任务级语义 rerank
- rerank 主输入采用：
  - `sub_query_text`
  - `document_title`
  - `section_title`
  - `chunk_text`
- `fused_rrf_score` 仅作为辅助调试信息保留，不作为 rerank 主判断输入
- 每个 `sub-query` 保留：
  - `top 5`
