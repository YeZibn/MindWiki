# 8.4 BM25 检索字段与权重设计

## 任务目标

为个人知识库 RAG 项目第一阶段定义 BM25 检索的主对象、字段范围、字段权重优先级和命中来源表达方式，使系统既能支持关键词精确命中，也能与向量召回形成稳定互补。

本子任务对应路线图中的 `8.4`：

- 设计 BM25 检索字段与权重，明确标题、标签、正文的纳入范围和命中来源表达

## 用户确认结论

基于当前讨论，`8.4` 采用以下正式设计：

- 第一阶段以 `chunk` 作为 BM25 检索主对象
- 纳入第一阶段 BM25 检索的字段包括：
  - `document_title`
  - `section_title`
  - `document_tags`
  - `chunk_text`
- 第一阶段不纳入主 BM25 字段的内容包括：
  - `keywords`
  - `chunk_summary`
  - `entities`
- 字段权重优先级采用：
  - `document_title > section_title > document_tags > chunk_text`
- `document_tags` 直接作为 `chunk` 级 BM25 字段参与索引
- 检索结果应保留 `match_sources`，用于表达命中来源

## 一、BM25 在第一阶段的作用

BM25 检索用于解决以下问题：

- 精确命中专有名词、术语、标题词
- 支持明显关键词驱动的查找请求
- 作为向量召回的补充路径，弥补语义召回在精确字面命中上的不足

因此，第一阶段检索层正式采用：

- 向量召回负责语义相似
- BM25 负责关键词精确匹配

## 二、主对象与字段范围

第一阶段 BM25 检索主对象正式确定为：

- `chunk`

第一阶段建议将以下字段纳入 `chunk` 级 BM25 检索：

1. `document_title`
2. `section_title`
3. `document_tags`
4. `chunk_text`

### 1. `document_title`

- 纳入 BM25
- 权重最高

### 2. `section_title`

- 纳入 BM25
- 权重次高

### 3. `document_tags`

- 纳入 BM25
- 权重高于 `chunk_text`
- 不单独作为主检索对象，而是作为 `chunk` 级字段参与索引

### 4. `chunk_text`

- 纳入 BM25
- 作为基础主体字段参与索引
- 权重低于标题与标签字段

## 三、第一阶段不纳入 BM25 的字段

第一阶段正式不将以下内容纳入主 BM25 字段：

1. `keywords`
2. `chunk_summary`
3. `entities`

原因：

- 关键词稳定性不够强，且容易重复放大正文词
- 摘要是派生结果，不是原始证据
- 实体第一阶段尚未建立复杂消歧体系

## 四、字段权重优先级

第一阶段正式采用如下权重优先级：

```text
document_title > section_title > document_tags > chunk_text
```

## 五、标签进入 BM25 的方式

第一阶段正式采用：

- `document_tags` 直接作为 `chunk` 级 BM25 字段参与索引

这意味着：

- 文档标签会被带入该文档下所有 chunk 的 BM25 检索语料
- 标签既可作为精确命中信号，也可作为后续过滤与排序辅助信号

## 六、命中来源表达

第一阶段正式要求 BM25 检索结果保留：

- `match_sources`

建议至少支持以下来源标识：

- `document_title`
- `section_title`
- `document_tags`
- `chunk_text`

## 七、最终结论

`8.4` 第一阶段正式采用以下方案：

- 第一阶段以 `chunk` 作为 BM25 检索主对象
- 纳入 BM25 的字段包括：
  - `document_title`
  - `section_title`
  - `document_tags`
  - `chunk_text`
- 第一阶段不纳入主 BM25 字段的内容包括：
  - `keywords`
  - `chunk_summary`
  - `entities`
- 字段权重优先级采用：
  - `document_title > section_title > document_tags > chunk_text`
- `document_tags` 直接作为 `chunk` 级 BM25 字段参与索引
- 检索结果保留 `match_sources`，用于表达命中来源
