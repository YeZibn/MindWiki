# 设计文档目录说明

`doc/design` 用于存放个人知识库 RAG 项目的设计文档。

目录约定如下：

- 每个 Step 使用独立子目录
- 子目录命名格式为 `step-xx-topic`
- 子任务文档命名格式为 `xx.yy-task-name.md`
- 子任务文件名必须显式包含编号，便于引用、讨论与后续拆分

当前建议目录结构示例：

- `step-01-product-scope`
- `step-02-system-architecture`
- `step-03-domain-model`
- `step-04-storage-design`
- `step-05-ingestion`
- `step-06-preprocessing`
- `step-07-chunking-and-enrichment`
- `step-08-index-and-retrieval`
- `step-09-retrieval-orchestration`
- `step-10-generation`
- `step-11-frontend-mvp`
- `step-12-evaluation`
- `step-13-knowledge-management-features`
- `step-14-engineering-foundation`
- `step-15-llm-module-and-model-service`
