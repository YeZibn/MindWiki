# 5.5 导入错误分类与 Warning 分类

## 任务目标

定义个人知识库 RAG 项目第一阶段导入流程中的错误分类、跳过原因、warning 分类与可重试标记，为导入任务状态、CLI 输出、日志排查和失败重试提供统一标准。

本子任务对应路线图中的 `5.5`：

- 设计导入错误分类，例如格式不支持、解析失败、存储失败

## 关联文档

- `05.02-import-job-payload-and-state-machine.md`
- `05.03-markdown-and-pdf-adapter-protocol.md`
- `05.04-file-deduplication-strategy.md`

## 用户确认结论

基于当前讨论，`5.5` 采用以下正式设计：

- 错误信息采用三层结构：
  - `error_type`
  - `reason_code`
  - `message`
- 第一阶段定义统一错误大类
- `skipped` 也必须有明确 `reason_code`
- warning 与 error 分开
- 每类错误应标记是否可重试：
  - `retryable: true`
  - `retryable: false`

## 错误信息结构

第一阶段错误信息建议采用如下结构：

```json
{
  "error_type": "parse_error",
  "reason_code": "pdf_text_extraction_failed",
  "message": "PDF 文本抽取失败，当前版本仅有限支持扫描版 PDF。",
  "retryable": false
}
```

### 字段说明

| 字段 | 说明 |
| --- | --- |
| `error_type` | 错误大类，用于系统分类处理 |
| `reason_code` | 具体原因码，用于机器判断和统计 |
| `message` | 面向用户或日志的可读描述 |
| `retryable` | 是否适合重试 |

## 第一阶段错误大类

第一阶段建议定义以下错误大类：

1. `file_error`
2. `adapter_error`
3. `parse_error`
4. `dedup_error`
5. `storage_error`
6. `embedding_error`
7. `index_error`
8. `system_error`

## 1. `file_error`

### 说明

表示文件访问、文件状态或文件基础信息异常。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `file_not_found` | 文件不存在 | false |
| `file_permission_denied` | 无权限读取文件 | false |
| `empty_file` | 文件为空 | false |
| `file_read_failed` | 文件读取失败 | true |

## 2. `adapter_error`

### 说明

表示文件类型识别或接入适配阶段失败。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `unsupported_file_type` | 文件类型不支持 | false |
| `file_type_detection_failed` | 文件类型识别失败 | false |
| `adapter_output_invalid` | 适配器输出不符合规范 | false |

## 3. `parse_error`

### 说明

表示 Markdown / PDF 内容解析失败。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `markdown_parse_failed` | Markdown 解析失败 | false |
| `frontmatter_parse_failed` | frontmatter 解析失败 | false |
| `pdf_text_extraction_failed` | PDF 文本抽取失败 | false |
| `pdf_scanned_without_ocr` | PDF 为扫描版，当前无法抽取文本 | false |
| `encoding_error` | 编码异常 | false |

## 4. `dedup_error`

### 说明

表示去重、hash 计算或来源判断阶段失败。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `content_hash_failed` | 内容 hash 计算失败 | true |
| `source_resolution_failed` | 来源解析失败 | true |
| `dedup_check_failed` | 去重判断失败 | true |

## 5. `storage_error`

### 说明

表示 PostgreSQL 或本地文件系统写入失败。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `postgres_write_failed` | PostgreSQL 写入失败 | true |
| `postgres_read_failed` | PostgreSQL 读取失败 | true |
| `object_storage_write_failed` | 原始文件存储失败 | true |
| `object_storage_read_failed` | 原始文件读取失败 | true |

## 6. `embedding_error`

### 说明

表示 embedding 生成阶段失败。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `embedding_model_call_failed` | embedding 模型调用失败 | true |
| `embedding_response_invalid` | embedding 返回结果无效 | true |
| `embedding_dimension_mismatch` | embedding 维度不匹配 | false |

## 7. `index_error`

### 说明

表示 Qdrant 或 BM25 检索索引写入失败。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `qdrant_write_failed` | Qdrant 写入失败 | true |
| `qdrant_delete_failed` | Qdrant 删除或失效处理失败 | true |
| `bm25_index_update_failed` | BM25 检索索引更新失败 | true |

## 8. `system_error`

### 说明

表示系统内部未预期错误。

### 常见 reason code

| reason_code | 说明 | retryable |
| --- | --- | --- |
| `unexpected_error` | 未预期异常 | true |
| `task_interrupted` | 任务中断 | true |
| `timeout` | 任务超时 | true |

## `skipped` reason code

`skipped` 表示任务无需继续处理，但不是异常失败。

第一阶段建议定义以下跳过原因：

| reason_code | 说明 |
| --- | --- |
| `unsupported_file_type` | 文件类型不支持 |
| `empty_file` | 文件为空 |
| `content_unchanged` | 内容 hash 未变化 |
| `excluded_by_rule` | 被导入规则排除 |

## Warning 分类

warning 表示导入可以继续，但存在需要提醒用户或后续排查的信息。

### Warning 与 Error 的区别

| 类型 | 是否阻断导入 | 是否影响任务状态 |
| --- | --- | --- |
| error | 是 | 通常导致 `failed` |
| warning | 否 | 通常仍可 `success` |

## 第一阶段 warning code

| warning_code | 说明 |
| --- | --- |
| `duplicate_content` | 不同来源下出现相同内容 |
| `source_conflict` | 同路径下出现疑似来源冲突 |
| `possible_replacement` | 文件可能被替换为另一份文档 |
| `structure_changed` | 文档结构明显变化 |
| `title_changed_significantly` | 文档标题明显变化 |
| `low_text_quality` | 文本抽取质量较低 |
| `metadata_incomplete` | 元数据不完整 |

## Retryable 判断原则

### 可重试错误

通常包括：

- 临时读写失败
- 模型调用失败
- Qdrant 写入失败
- BM25 检索索引更新失败
- 任务中断
- 超时

### 不可重试错误

通常包括：

- 文件不存在
- 文件类型不支持
- 文件为空
- 扫描版 PDF 无 OCR 支持
- embedding 维度不匹配

### 设计原则

- 可重试不代表必须自动重试
- 第一阶段最多重试 2 次
- 超过重试次数仍失败时应明确反馈给用户

## CLI 输出建议

### failed 输出

建议包含：

- 任务 ID
- 文件路径
- `error_type`
- `reason_code`
- `message`
- 是否可重试
- 已重试次数

### skipped 输出

建议包含：

- 任务 ID
- 文件路径
- `reason_code`
- 简短说明

### warning 输出

建议包含：

- 文件路径
- `warning_code`
- 简短说明
- 是否影响导入结果

## 与任务状态的关系

### `failed`

由 error 驱动。

### `skipped`

由 skipped reason 驱动。

### `success`

即使存在 warning，只要核心导入成功，任务仍可 `success`。

## 最终结论

第一阶段导入错误体系采用“错误大类 + 原因码 + 用户消息 + 可重试标记”的结构：

- error 用于失败状态
- skipped 用于非异常跳过
- warning 用于不阻断导入的风险提示
- retryable 用于辅助重试策略
