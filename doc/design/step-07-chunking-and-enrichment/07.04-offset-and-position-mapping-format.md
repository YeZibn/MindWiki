# 7.4 偏移量与位置映射格式设计

## 任务目标

定义第一阶段 section 与 chunk 的位置映射格式，明确系统内部如何基于标准化文本偏移量追踪内容位置，以及对外如何用 Markdown 行号、PDF 页码与页内块序号等方式为检索和引用提供可理解的定位信息。

本子任务对应路线图中的 `7.4`：

- 设计 chunk 与原文偏移量的记录格式，支持前端高亮定位

## 一、设计目标

位置映射设计要同时满足以下目标：

- 系统内部切分、合并、重叠时有稳定依据
- 引用展示时用户能看懂定位结果
- 检索结果能回跳到原文具体部分
- 不因为格式差异而丢失追溯能力

## 二、位置映射的四层结构

第一阶段建议把位置映射拆成 4 层：

1. 原始文件层
2. 标准化文本层
3. 结构层
4. 用户可理解定位层

## 1. 原始文件层

这一层用于指向原始来源文件本身。

建议至少记录：

- `source_id`
- `document_id`
- `original_path`
- `source_type`

作用：

- 确保任何 section / chunk 都能追溯到原始来源
- 支撑后续原文件跳转

## 2. 标准化文本层

这一层是系统内部最核心的位置基础。

第一阶段正式建议：

- 内部统一以标准化文本 `offset` 为主映射基础

建议记录：

- `start_offset`
- `end_offset`

作用：

- 支撑 section 切分
- 支撑 chunk 切分
- 支撑 overlap 生成
- 支撑 chunk 合并与再拆分

说明：

- offset 是内部处理最稳定的统一基准
- 不同来源类型最终都可收敛到标准化文本偏移量

## 3. 结构层

这一层用于表达内容在结构树中的位置。

建议记录：

- `section_id`
- `parent_section_id`
- `chunk_id`
- `chunk_index`

作用：

- 确定 chunk 属于哪个 section
- 支撑引用展示中的结构路径
- 支撑检索结果中的上下文归属

第一阶段正式要求：

- section 和 chunk 都必须记录位置映射
- 不能只给 chunk 记录位置，section 也需要可定位

## 4. 用户可理解定位层

这一层用于最终展示给用户的可读定位信息。

不同来源类型采用不同表现：

- Markdown：以行号为主
- PDF：以页码 + 页内块序号为主

## 三、Markdown 定位规则

### 1. 对外定位方式

Markdown 对外展示时，第一阶段以：

- `line_start`
- `line_end`

作为主要定位方式。

### 2. 原因

Markdown 文档最自然的人工定位单位就是行号。

这样做的好处：

- 便于回看原文
- 便于前端高亮
- 便于开发和调试

### 3. 建议字段

```json
{
  "line_start": 12,
  "line_end": 24
}
```

## 四、PDF 定位规则

### 1. 对外定位方式

PDF 对外展示时，第一阶段至少应提供：

- `page_start`
- `page_end`
- `page_block_start`
- `page_block_end`

即：

- 页码
- 页内块序号

### 2. 原因

PDF 如果只有文档级来源或只有页码，定位仍然过粗。

加入页内块序号后，至少可以定位到：

- 第几页
- 该页的第几个文本块附近

这更符合项目对引用准确性的要求。

### 3. 建议字段

```json
{
  "page_start": 3,
  "page_end": 4,
  "page_block_start": 2,
  "page_block_end": 5
}
```

## 五、PDF BBox 预留规则

第一阶段虽然不强依赖 PDF 坐标高亮，但正式建议预留 `bbox` 字段。

### 1. 设计理由

- 后续前端高亮更精确
- 后续 OCR 结果也可能提供坐标
- 未来 PDF 定位可以升级，而不需要推翻数据结构

### 2. 建议格式

```json
{
  "bbox": [
    {
      "page": 3,
      "x1": 72,
      "y1": 120,
      "x2": 520,
      "y2": 180
    }
  ]
}
```

### 3. 第一阶段策略

- 可以为空
- 不要求所有 PDF 都必须产出 bbox
- 但字段应预留

## 六、Section 与 Chunk 的位置映射要求

第一阶段正式要求：

- section 和 chunk 都必须具备位置映射

原因：

- section 需要支持“来自哪一节”的展示
- chunk 需要支持“来自该节中的哪一段”的展示
- 两层都缺一不可

## 七、Chunk 跨块映射规则

chunk 可以跨多个 PDF page block，但默认不跨 section。

正式策略：

- 允许一个 chunk 对应多个 page block
- 允许一个 chunk 跨页
- 但默认不跨 section

这意味着：

- 位置映射不能假设一个 chunk 只对应一个点
- 应支持一个 chunk 映射到一组位置片段

## 八、建议统一位置映射格式

第一阶段建议定义统一结构，例如：

```json
{
  "source_id": "uuid",
  "document_id": "uuid",
  "section_id": "uuid",
  "chunk_id": "uuid",
  "source_type": "markdown | pdf",
  "start_offset": 1200,
  "end_offset": 1680,
  "line_start": 40,
  "line_end": 58,
  "page_start": 3,
  "page_end": 4,
  "page_block_start": 2,
  "page_block_end": 5,
  "segments": [
    {
      "start_offset": 1200,
      "end_offset": 1360,
      "page": 3,
      "page_block_index": 2
    },
    {
      "start_offset": 1361,
      "end_offset": 1680,
      "page": 4,
      "page_block_index": 1
    }
  ],
  "bbox": []
}
```

字段说明：

- `start_offset` / `end_offset`：内部统一基准
- `line_start` / `line_end`：Markdown 对外定位
- `page_start` / `page_end`：PDF 页定位
- `page_block_start` / `page_block_end`：PDF 页内块定位
- `segments`：支持一对多映射
- `bbox`：未来更细粒度高亮预留

## 九、Section 建议位置字段

section 建议至少记录：

```json
{
  "section_id": "uuid",
  "document_id": "uuid",
  "start_offset": 800,
  "end_offset": 2200,
  "line_start": 10,
  "line_end": 60,
  "page_start": 2,
  "page_end": 4
}
```

## 十、Chunk 建议位置字段

chunk 建议至少记录：

```json
{
  "chunk_id": "uuid",
  "section_id": "uuid",
  "chunk_index": 3,
  "start_offset": 1200,
  "end_offset": 1680,
  "line_start": 40,
  "line_end": 58,
  "page_start": 3,
  "page_end": 4,
  "page_block_start": 2,
  "page_block_end": 5
}
```

## 十一、与引用展示的关系

最终引用展示建议：

- Markdown：优先显示行号
- PDF：优先显示页码与页内块序号

例如：

```text
《RAG 设计笔记》 / 检索层 / 第 40-58 行
《学习资料汇编》 / 第 3 页 / 第 2 文本块
```

如果未来支持 bbox，则可进一步支持：

- PDF 原文高亮
- 精确框选区域跳转

## 十二、最终结论

`7.4` 正式采用以下位置映射策略：

- 系统内部统一以标准化文本 offset 作为主映射基础
- Markdown 对外以行号为主定位
- PDF 对外至少提供页码 + 页内块序号
- 第一阶段预留 PDF bbox 字段，但不强依赖
- section 和 chunk 都必须记录位置映射
- chunk 可以跨多个 page block，但默认不跨 section
- 位置映射结构必须支持一对多片段映射

该设计将直接支撑后续检索结果展示、回答引用定位、前端高亮跳转和工程调试能力。
